\subsection{Serialization を用いたプログラム}
\begin{minted}[frame=single]{kotlin}
    import kotlinx.serialization.Serializable
    import kotlinx.serialization.json.Json

    @Serializable
    data class Task(
        // 省略
    )

    fun main() {
        val task = Task(
            id = 0,
            name = "Learn Serialization"
        )
        val json = Json.encodeToString(task)
        println("Encoded: $json")

        val decoded = Json.decodeFromString<Task>(json)
        println("Decoded: $decoded")
    }
\end{minted}

出力結果は，
\begin{minted}[frame=lines]{bash}
    Encoded: {"id":0,"name":"Learn Serialization"}
    Decoded: Task(id=0, name=Learn Serialization, done=false)
\end{minted}

因みに，\texttt{./gradlew dependencies}は，\texttt{No configurations}を返した．

上のプログラムでは，プログラム実行中のメモリに各\texttt{task}が保持されている．project directory内にjsonファイルを作成し，そのファイルへの書き込み/書き出しを実装してみた．
\begin{minted}[frame=single]{kotlin}
    import kotlinx.serialization.Serializable
    import kotlinx.serialization.json.Json
    import java.io.File

    // annotation, data class の定義は省略

    fun main() {
        val file = File("tasks.json")
        val tasks = if (file.exists()) {
            Json.decodeFromString<MutableList<Task>>(file.readText())
        } else {
            mutableListOf()
        }

        val addedTask = Task(
            id = 0,
            name = "Learn Serialization"
        )
        tasks.add(addedTask)

        file.writeText(Json.encodeToString(tasks))
        println(tasks)
    }
\end{minted}
出力は，
\begin{minted}[frame=lines]{bash}
    [Task(id=0, name=Learn Serialization, done=false)]
\end{minted}
前回と出力形式が変わっている．前回は\texttt{println(task)}(\texttt{task}は\texttt{Task}型)，今回はjson ファイルを書き出してそのまま出力している．\texttt{Json.decodeFromString<MutableList<Task>>}から，\texttt{Task}型のリストとして出力されたと考えられる．
また，\texttt{file=File("task.json)}はファイル名ではなくpath 名を表している．このpathが\texttt{.../app/src}の下から始まっていると予想して，pathを\texttt{"main/kotlin/tesks.json"}とするとエラーが出た．続いて，\texttt{"src/main/kotlin/tasks.json"}とすると正常に動作した．よってこの部分のpathは\texttt{.../app}の下から始まっていることが分かった．

続いて，書き込み/書き出しを分けて実装して，build.gradle.kts/application でのmainClass の切り替えでtasks.json を操作した．
.../example/AddTask.kt およびListTasks.kt と，.../example/model/Task.kt を追加した．まず，Task.kt にはdata class の定義を書いた．このファイルにだけannotation \texttt{@Serializable}を書く．
\begin{minted}[frame=single]{kotlin}
    import kotlinx.serialization.Serializable

    @Serializable
    data class Task(
        val id: Int,
        val name: String,
        val done: Boolean = false
    )
\end{minted}
addTask.kt には，ファイルの発見/作成と書き込みを実装する．
\begin{minted}[frame=single]{kotlin}
    import kotlinx.serialization.json.Json
    import java.io.File
    import org.example.model.Task

    fun main() {
        val file = File("src/main/kotlin/data/tasks.json")
        // 変更できる(mutable)リストとしてtask.json を書き出し
        val tasks = if (file.exists()) {
            Json.decodeFromString<MutableList<Task>>(file.readText())
        } else {
            mutableListOf()
        }

        val addedTask = Task(
                id = 1,
                name = "Clean the Room"
                )
        // リストの変更
        tasks.add(addedTask)
        // リスト → json ファイル
        file.writeText(Json.encodeToString(tasks))

        println("Added. Total: ${tasks.size}")
    }
\end{minted}

ListTasks.kt には，ファイルの発見と書き出しを実装する．
\begin{minted}[frame=single]{kotlin}
    import kotlinx.serialization.json.Json
    import org.example.model.Task
    import java.io.File

    fun main() {
        val file = File("src/main/kotlin/data/tasks.json")
        if (!file.exists()) {
            println("No tasks found.")
            return
        }

        // 読込だけの(mutable でない)リストとしてtasks.json を書き出し
        val tasks = Json.decodeFromString<List<Task>>(file.readText())
        // tasks の各要素をit として受ける
        tasks.forEach { println("[${it.id}] ${it.name} - done: ${it.done}") }
    }
\end{minted}
